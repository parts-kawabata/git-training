# 第05章「要件定義、その前に」感想文
1. この章の要点（自分の言葉で3行）：ソフトウェア開発における要件定義とは、「UI」「機能」「データ」の3つの要素を定めていくこと。では、「UIが決まった」というゴールの状態を達成するためには、何をどうすればよいのか。そのときに必要な材料を揃えるための準備作業を、要件定義の前に行わなければならない。
2. なるほど／腑に落ちた点（1〜2個）：要件定義を行うために必要な材料を揃える作業がある。
3. 身近な例で言いかえると（教本の例、または「入金消込」のような“想像でわかる”例でOK。実際の自社業務は知らなくてよい）：目玉焼きという成果を出すには卵という材料が必要なのと同様に、定義されたUIという成果を出すにも材料が必要になる。その材料を揃えるための、要件定義を作るうえでの準備作業をしなければならない。
4. 疑問・もっと知りたい点（1個）：準備作業をしなかった場合、どのようなことが起きてしまうのか
5. 前の章・V字（開発工程）とのつながり（1行）：要件を定めるための事前準備

①作り始める前に、何を決めておくと後で楽になる？（“やらないこと”の線引き含む）
- まず「目的・ゴール」「完成の合格ライン」「対象ユーザーは誰か」「現状と課題」などの土台を決めておかなければならず、これがないと全部ブレてしまう。「何のために作るか」が定まると、迷ったとき常にここへ立ち返って判断でき、「どうなったら完成か」を先に握ることができる。また、UIも機能も「誰が使うか」で正解が変わるため、ここがブレてしまうと全てやり直しになってしまう。そして、今の困りごとを言語化しておくと、機能の要否をここで判定することができる。
